# DeltaV FHX Nameset Editor 使用说明

## 概述

本工具用于批量翻译/编辑 DeltaV FHX 配置文件中的 nameset 值。面向 Emerson DeltaV 系统工程师，解决中文 nameset 值导致的 DeltaV 导入字符集错误。支持任何 FHX 类型（Library、Control Strategies、Setup、Recipes 等），自动识别并处理。

### 支持的翻译内容

| 类型 | 说明 | 示例 |
|------|------|------|
| ENUMERATION_SET 定义 | 值名称翻译 | `VALUE=1 NAME="停止中"` → `VALUE=1 NAME="Stopped"` |
| STRING_VALUE 引用 | 字段引用值翻译 | `STRING_VALUE="正在运行"` → `STRING_VALUE="Running"` |
| 表达式引用 | 公式中的值翻译 | `'$phase_state:已保留'` → `'$phase_state:Held'` |
| SYSTEM_ALARM / USER_ALARM | 报警块字段翻译 | DESCRIPTION、ALARM_WORD、MESSAGE、CATEGORY |
| PRIORITY_NAME | 优先级名称翻译 | `"危急"` → `"CRITICAL"` |
| LOCALE | 语言区域替换 | 根据 New Database.fhx 自动替换 |

### 工作流程

```
Step 1: 对比任意 FHX 与 New Database.fhx → 导出 Excel（自动填充英文建议值）
    ↓
用户在 Excel 中审核/修改 "New Value" 列
    ↓
Step 2: 读取 Excel → 生成新的 FHX 文件（后缀 _NEW）
```

## 使用步骤

### 第一步：准备文件

你需要以下文件：

1. **待翻译的 FHX 文件** — Library、Control Strategies、Setup、Recipes 或其他需要迁移的 FHX（工具自动识别类型）
2. **New Database.fhx** — 目标 DeltaV 版本导出的 New Database 文件（作为英文参考源）

> New Database.fhx 提供标准英文 nameset 定义，工具会自动匹配对应值。

### 第二步：对比并导出 Excel

1. 双击运行 `DeltaV_FHX_Nameset_Editor.exe`
2. 在 **Step 1** 区域：
   - **FHX File**：选择待翻译的 FHX 文件
   - **New Database.fhx**：选择目标版本的 New Database 文件
   - 点击 **Compare and Export Excel**
3. 等待处理完成，日志区域会显示对比结果摘要

生成的 Excel 文件保存在 FHX 文件同目录，文件名后缀 `_comparison.xlsx`。

### 第三步：审核 Excel

打开生成的 Excel，包含以下工作表：

#### Namesets — ENUMERATION_SET 定义

| 列 | 说明 |
|----|------|
| SET Name | nameset 名称 |
| Category | 分类 |
| Status | `Both` = 两端都有，`FHX only` = 仅 FHX 中存在 |
| FHX Values | 当前 FHX 中的值列表 |
| New Value | **需要填写** — 目标值 |
| Description | nameset 描述 |

#### String Values — STRING_VALUE 引用

| 列 | 说明 |
|----|------|
| SET Name | nameset 名称 |
| Current Value | 当前引用的值 |
| Count | 在文件中出现的次数 |
| New Value | **需要填写** — 目标值 |
| Setup Entries | New Database 中对应的所有条目（供参考） |

#### Expression Refs — 表达式引用

| 列 | 说明 |
|----|------|
| SET Name | nameset 名称 |
| Current Value | 表达式中的当前值 |
| Count | 出现次数 |
| New Value | **需要填写** — 目标值 |
| Setup Entries | New Database 中对应的所有条目 |

#### Alarm Types — 报警块

| 列 | 说明 |
|----|------|
| Alarm Name | 报警名称 |
| Status | `Both` / `Old only` / `New only` |
| Old Description / New Description | 描述字段对比 |
| Old Alarm Word / New Alarm Word | 报警字对比 |
| Old Message / New Message | 消息字段对比 |
| Old Category / New Category | 分类字段对比 |

#### Alarm Priorities — 优先级名称

| 列 | 说明 |
|----|------|
| Old Priority Name | 当前中文优先级名 |
| New Priority Name | **需要填写** — 目标英文名 |

### New Value 列填写规则

- **简单填写**：直接写名称，如 `STOP`、`Running`
  - 工具自动转换为完整格式 `VALUE=1 NAME="STOP"`
- **完整填写**：`VALUE=1 NAME="STOP"`
- **留空**：该行不修改，保持原样

> 蓝色底色 = 已有自动建议（可直接使用），黄色底色 = 需手动填写。

### 第四步：生成新 FHX

1. 在 **Step 2** 区域：
   - **FHX File**：选择原 FHX 文件（与 Step 1 相同）
   - **New Database.fhx**：选择 New Database 文件（与 Step 1 相同）
   - **Edited Excel**：选择上一步编辑后的 Excel 文件
2. 点击 **Generate New FHX**
3. 工具会先验证 Excel 数据格式，如有问题会提示并阻止生成
4. 生成的新文件保存在原文件同目录，文件名后缀 `_NEW`
   - 例：`Library_Migrated.fhx` → `Library_Migrated_NEW.fhx`

### 第五步：导入 DeltaV

将 `_NEW.fhx` 导入 DeltaV 系统，验证导入无错误。

## 内建标准 DeltaV nameset 自动翻译

以下标准 nameset 的中英文映射已内建，工具会自动填充建议值：

| Nameset | 包含值示例 |
|---------|-----------|
| `$phase_state` | 停止中→Stopping, 正在运行→Running, 已完成→Complete |
| `$recipe_state` | 空闲→Idle, 正在启动→Starting, 已退出→Aborted |
| `$sfc_action_states` | 激活→Active, 不活动→Inactive, 失败→Failed |
| `$phase_command` | 启动→Start, 停止→Stop, 保持→Hold, 复位→Reset |
| `$sfc_commands` | 启动顺控→Start Sequence, 停止顺控→Stop Sequence |
| `$sfc_states` | 顺控空闲→Sequence Idle, 顺控激活→Sequence Active |
| `$dc_states` | 锁定→Locked, 已跳车→Tripped |
| `$module_states` | 离线→Out Of Service, 服务中→In Service |
| `$time` | 秒→Seconds, 分→Minutes, 时→Hours |
| `$isel_typ` | 第一个有效值→First Valid, 最小→Minimum, 最大→Maximum |
| 其他 | `$sysstat_opts`, `$time_format`, `$phase_restart_types` 等 |

## CLI 命令行模式

```bash
# 对比并导出 Excel
python fhx_migrator.py compare <任意FHX文件> --setup <New Database.fhx> [-o <输出路径>]

# 从编辑后的 Excel 生成新 FHX
python fhx_migrator.py generate <任意FHX文件> --setup <New Database.fhx> --excel <Excel文件> [-o <输出路径>]
```

| 参数 | 说明 |
|------|------|
| `compare` | 子命令：对比任意 FHX 与 New Database，导出 Excel |
| `generate` | 子命令：读取 Excel 生成新 FHX |
| `--setup` | 必填。New Database.fhx 参考文件路径 |
| `--excel` | `generate` 必填。编辑后的 Excel 文件路径 |
| `-o, --output` | 可选。输出文件路径 |

> 注意：旧版的 `cs-compare`、`cs-generate`、`lib-compare`、`lib-generate` 子命令已移除，统一使用 `compare` / `generate`，工具自动识别 FHX 类型。

**示例：**

```bash
# 对比（适用于任何 FHX 类型）
python fhx_migrator.py compare Library_Migrated.fhx --setup New_Database.fhx

# 生成（Excel 在上一步已编辑）
python fhx_migrator.py generate Library_Migrated.fhx --setup New_Database.fhx --excel Library_Migrated_comparison.xlsx

# 指定输出路径
python fhx_migrator.py generate Control_Strategies.fhx --setup New_Database.fhx --excel edited.xlsx -o D:\output\CS_EN.fhx
```

## 常见问题

### Q: 导入后仍有中文错误？
A: 检查 Excel 中是否有遗漏未填的行。标准 DeltaV nameset（如 `$phase_state`）已内置自动转换，通常无需手动填写。

### Q: New Value 列格式？
A: 最简方式是直接写英文名称（如 `STOP`），工具自动补全 `VALUE=...` 编号。

### Q: 能否只翻译部分 nameset？
A: 可以。在 Excel 中只填写需要翻译的行，留空的行保持原样。

### Q: Excel 文件打不开？
A: 确保 Excel 已关闭该文件。工具在读取时会自动重试，但如果 Excel 持续锁定文件会导致失败。

### Q: FHX 文件编码？
A: FHX 文件使用 UTF-16 LE with BOM 编码，工具自动处理读写编码，输出保持相同格式。

### Q: 生成前 Excel 验证失败？
A: 工具会在生成前检查 Excel 数据格式，如 VALUE 编号缺失、名称格式不正确等。根据提示修复对应行后重新生成。

### Q: 旧版 CLI 命令不工作？
A: 旧版的 `cs-compare`、`cs-generate`、`lib-compare`、`lib-generate` 已合并为 `compare` / `generate`。只需使用这两个命令，工具会自动处理所有 FHX 类型。

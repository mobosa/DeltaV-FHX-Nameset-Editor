# DeltaV FHX Nameset Editor 使用说明

## 概述

本工具用于批量编辑 DeltaV FHX 配置文件中的 nameset 值。典型用途是将中文 nameset 值翻译为英文，解决 DeltaV 系统不支持中文字符导致的导入错误。也可用于英文环境下的 nameset 值批量修改。

### 支持的文件类型

| 标签页 | 输入文件 | 处理内容 |
|--------|---------|---------|
| Setup | Setup.fhx | ENUMERATION_SET 定义（值名称） |
| Library | Library FHX | ENUMERATION_SET 定义 + STRING_VALUE 引用 + 表达式引用 |
| Control Strategies | CS FHX | STRING_VALUE 引用 |

### 工作流程

每个标签页都遵循相同两步流程：

```
Step 1: 对比 → 导出 Excel（自动填充英文建议值）
    ↓
用户在 Excel 中审核/修改 "New Value" 列
    ↓
Step 2: 生成 → 读取 Excel → 输出新的英文 FHX 文件
```

## 使用步骤

### 第一步：准备文件

确保你有以下文件：

1. **Setup.fhx** — 目标 DeltaV 版本的 Setup 导出文件（作为翻译/修改参考）
2. **Migrated或者需要编辑的 FHX 文件** — Library、Control Strategies 等

> Setup.fhx 是参考源，工具会从中自动匹配对应的 nameset 值/Local信息 。

### 第二步：对比并导出 Excel

1. 双击运行 `DeltaV_FHX_Nameset_Editor.exe`
2. 选择对应的标签页（Library / Control Strategies）
3. 在 **Step 1** 区域：
   - **FHX File**：选择Migrated FHX 文件
   - **Setup.fhx**：选择新数据库导出的 Setup 文件
   - 点击 **"Compare and Export Excel"**
4. 等待处理完成，工具会自动打开 Excel 文件位置

### 第三步：审核 Excel

生成的 Excel 文件包含以下工作表：

#### Namesets（Library 标签页）
| 列 | 说明 |
|----|------|
| Set Name | ENUMERATION_SET 名称 |
| Current Value | 当前值（如 `VALUE=1 NAME="停止中"`） |
| Count | 在文件中出现的次数 |
| New Value | **需要填写** — 目标值 |
| Setup Entries | Setup 中对应的完整条目列表 |

#### String Values（Library / CS 标签页）
| 列 | 说明 |
|----|------|
| SET Name | nameset 名称 |
| Current Value | 当前 STRING_VALUE 引用的值 |
| Count | 出现次数 |
| New Value | **需要填写** — 目标值 |

#### Expression Refs（Library 标签页）
| 列 | 说明 |
|----|------|
| SET Name | nameset 名称 |
| Current Value | 表达式中引用的值 |
| Count | 出现次数 |
| New Value | **需要填写** — 目标值 |

#### 填写规则

- **简单填写**：直接写名称，如 `STOP`、`Running`
  - 工具会自动转换为完整格式 `VALUE=1 NAME="STOP"`
- **完整填写**：写完整格式 `VALUE=1 NAME="STOP"`
- **留空**：该行不做编辑，保持原样

> 工具会根据 Setup.fhx 中的同名 nameset 自动建议值（蓝色底色）。如果没有自动建议（黄色底色），需要手动填写。

### 第四步：生成新 FHX

1. 回到工具界面，切换到 **Step 2** 区域
2. 选择：
   - **FHX File**：Migrated FHX 文件
   - **Setup.fhx**：新数据库导出的 Setup 文件
   - **Edited Excel**：上一步审核/编辑后的 Excel 文件
3. 点击 **"Generate New FHX"**
4. 生成的新文件保存在原文件同目录，文件名后缀为 `_NEW`
   - 例：`Library_Migrated_v1431.fhx` → `Library_Migrated_v1431_NEW.fhx`

### 第五步：导入 DeltaV

将生成的 `_NEW.fhx` 文件导入 DeltaV 系统，检查是否还有导入错误。

## 三个标签页的区别

### Setup 标签页
- 处理 ENUMERATION_SET **定义**中的值名称
- 例：`VALUE=1 NAME="停止中"` → `VALUE=1 NAME="Stopped"`
- 输入：中文 Setup FHX + 英文 Setup FHX

### Library 标签页
- 处理三种中文内容：
  1. **ENUMERATION_SET 定义**（同 Setup）
  2. **STRING_VALUE 引用**：`SET="模块名" ... STRING_VALUE="中文值"` 中的值
  3. **表达式引用**：`'$phase_state:正在保持'` 等表达式中的中文值
- 额外处理：
  - **报警翻译**：SYSTEM_ALARM / USER_ALARM 块中的 DESCRIPTION、ALARM_WORD、MESSAGE、CATEGORY
  - **优先级翻译**：中文优先级名自动映射（"危急"→"CRITICAL"、"警告"→"WARNING"、"提示"→"ADVISORY"、"记录"→"LOG"）
  - **LOCALE 替换**：根据 Setup.fhx 替换 locale 字符串
- 标准 DeltaV nameset（如 `$phase_state`、`$sfc_action_states`）会自动填入，无需手动填写
- 输入：中文 Library FHX + 英文 Setup FHX

### Control Strategies 标签页
- 仅处理 **STRING_VALUE 引用**
- 输入：中文 CS FHX + 英文 Setup FHX

## CLI 命令行模式

工具支持命令行模式，不带参数运行启动 GUI，带参数进入 CLI 模式。

### Setup 模式

```bash
# 对比并导出 Excel
python fhx_migrator.py compare input.fhx --setup Setup.fhx [-o output.xlsx]

# 从编辑后的 Excel 生成新 FHX
python fhx_migrator.py generate input.fhx --setup Setup.fhx --excel edited.xlsx [-o output.fhx]
```

### Library 模式

```bash
python fhx_migrator.py lib-compare Library.fhx --setup Setup.fhx [-o output.xlsx]
python fhx_migrator.py lib-generate Library.fhx --setup Setup.fhx --excel edited.xlsx [-o output.fhx]
```

### Control Strategies 模式

```bash
python fhx_migrator.py cs-compare CS.fhx --setup Setup.fhx [-o output.xlsx]
python fhx_migrator.py cs-generate CS.fhx --excel edited.xlsx [--setup Setup.fhx] [-o output.fhx]
```

> 不指定 `-o` 时，输出文件自动生成在原文件同目录下。

## 常见问题

### Q: 导入后仍有中文错误怎么办？
A: 检查 Excel 中是否有遗漏未填的行。部分标准 DeltaV nameset（如 `$phase_state`）已内置自动转换，无需手动填写。

### Q: New Value 列的格式是什么？
A: 最简单的方式是直接写英文名称（如 `STOP`），工具会自动处理 VALUE 编号。

### Q: 生成的 Excel 在哪里？
A: 与输入的 FHX 文件在同一目录，文件名后缀为 `_nameset_comparison.xlsx` 或 `_library_comparison.xlsx`。

### Q: 能否只翻译部分 nameset？
A: 可以。在 Excel 中只填写需要翻译的行，留空的行保持原样。

### Q: 标准 DeltaV nameset 是什么？
A: DeltaV 系统内置的 nameset（如 `$phase_state`、`$recipe_state`），不在 FHX 文件中定义。工具内置了这些 nameset 的中英文映射，会自动翻译。

## 技术细节

- FHX 文件编码：UTF-16 LE with BOM
- 输出文件保持与输入相同的编码格式
- 使用 `re.subn` 进行精确文本替换，不会修改 FHX 结构
- 表达式替换使用 nameset 名称 + 值名称的精确匹配
